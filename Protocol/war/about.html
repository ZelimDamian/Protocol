Java Protocol Implementation

Java Protocol Implementation for improving financial safety of online shopping.

Some of the many assumptions

  1. There is only one Merchant and his name is "Merchant".
  2. Prices are integers.
  3. The forms are filled correctly always.

A little manual

A Run-Through test:

  1. Go to the bank website. Wait for the list of customers to appear (this may take time due to AppEngine server load).
  2. Generate a payment in the bank by selecting a bank client from the list and filling the form with payee being "Merchant" and whatever price.
  3. Copy the JSON code of the payment.
  4. Go to merchants website and choose the desired product.
  5. Optional: Click "Verify" to visit CA's website to see verification of the product.
  6. Click buy in the list of products to the TTP website
  7. Product id is already in place now you only need to paste the copied JSON into the text area and press "Parse JSON".
  8. The form is filled with the required data. Now press "Initiate protocol" button.
  9. Notice the encrypted contents of the product. Copy them somewhere.
  10. Optional: Click "Verify with CA" to make final verification.
  11. Click "Make payment"
  12. Use the received private key to decrypt the symmetric key.
  13. Use the decrypted symmteric key to decrypt the product contents.
